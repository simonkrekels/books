# book

A `beets`-style CLI for your academic PDF library. Import a paper, get clean metadata from Crossref / arXiv / Open Library, file it on disk by a path template you control, and search across the full text with a local vector index.

```
$ book import ~/Downloads/sakurai-modern-quantum-mechanics.pdf
Identified: Modern Quantum Mechanics (Sakurai & Napolitano, 2020)
ISBN: 9781108473224 · publisher: Cambridge University Press
[A]pply / [S]kip / [E]nter DOI / [Q]uit > a
imported #14 -> Sakurai/2020/modern-quantum-mechanics.pdf  (1843 chunks indexed)

$ book search "spin precession in a magnetic field" -k 3
1. Modern Quantum Mechanics  p.78   score=0.71
   ...the spin precesses about the field axis at the Larmor frequency omega_L = gamma B...
2. Quantum Computation and Quantum Information  p.174  score=0.64
   ...precession of a single qubit on the Bloch sphere is generated by sigma_z ...
```

## Why

Reference managers want to be the centre of your workflow. `book` does the opposite — it organises files on disk under a path template you choose, keeps the canonical bibliography in plain SQLite, and gives you a fast CLI to query and search the full text. Everything stays local. Anything else (LaTeX integration, RAG, sync) is something you bolt on top.

## Highlights

- **Import once, lookup automatic.** Sniffs DOI / arXiv ID / ISBN out of the PDF (metadata fields, front matter, and back matter — weighted so a body-cited DOI doesn't masquerade as the paper's own) and resolves it via Crossref, arXiv, or Open Library.
- **Files where you want them.** A `str.format` path template — `{author_last}/{year}/{title_slug}.pdf` by default — drives the on-disk layout. Move, copy, or symlink.
- **Full-text semantic search.** Every imported PDF is chunked, embedded with `BAAI/bge-small-en-v1.5`, and stored in a persistent Chroma index. `book search "your query"` returns ranked snippets with page numbers.
- **SQLite is the source of truth.** The vector index is a derived artefact: blow it away and `book reindex --all` rebuilds it from the PDFs.
- **Offline-friendly.** Flip `index.offline: true` and the embedder loads from local cache only — no Hugging Face round-trips, no progress-bar noise.
- **Single binary feel.** One `book` command, six subcommands, no daemon, no plugin host.

## Install

Requires Python 3.14+ and [`uv`](https://github.com/astral-sh/uv).

```bash
git clone git@github.com:simonkrekels/books.git
cd books
uv sync
uv run book --help
```

To use `book` from anywhere without `uv run`:

```bash
uv tool install --from . book
```

## Quickstart

```bash
# 1. Import a paper. Interactive by default — confirm the match before it commits.
uv run book import ~/Downloads/attention-is-all-you-need.pdf

# 2. Or batch-import a directory, auto-accepting the top match.
uv run book import --quiet ~/Downloads/papers/

# 3. List what's in the library.
uv run book ls --author Vaswani --year 2017

# 4. Show full metadata for a single entry.
uv run book show 1

# 5. Semantic search across every PDF you've imported.
uv run book search "scaled dot-product attention" -k 5

# 6. Re-chunk and re-embed (e.g. after swapping the embedder).
uv run book reindex --all
```

## Commands

| Command            | What it does                                                                 |
| ------------------ | ---------------------------------------------------------------------------- |
| `book import`      | Sniff identifier → fetch metadata → confirm → file the PDF → index it.       |
| `book ls`          | List with flags (`--author`, `--year`, `--journal`, `--tag`, `--doi`, `--isbn`) and free positional terms (LIKE-search title/abstract). |
| `book show`        | Pretty-print a single paper's full metadata.                                 |
| `book rm`          | Remove from the library. `--keep-file` is the default; pass `--delete-file` to also delete the PDF. |
| `book search`      | Embed a query, return top-`k` chunks across the index with page + snippet.   |
| `book reindex`     | Rebuild the vector index for `--all`, specific papers, or anything flagged `needs_reindex=1`. |

### Useful flags

```bash
book import --copy      paper.pdf      # default is move; --copy preserves the original
book import --quiet     papers/        # auto-accept top match for every PDF
book ls quantum --year 2024            # combine positional title-match + flag
book search "..." -k 10                # tune the result count
```

## Configuration

`book` ships sane defaults at `src/books/config_default.yaml`, then merges user overrides from the OS-appropriate confuse location (on macOS: `~/Library/Application Support/book/config.yaml`).

```yaml
library_dir: ~/Documents/papers
db_path: ~/.local/share/book/library.db
chroma_dir: ~/.local/share/book/chroma

import:
  mode: move                                           # copy | move | symlink
  path_template: "{author_last}/{year}/{title_slug}.pdf"

metadata:
  sources: [crossref, arxiv]
  user_agent: "book/0.1 (mailto:you@example.com)"      # mailto qualifies for Crossref's polite pool

index:
  model: BAAI/bge-small-en-v1.5
  chunk_tokens: 512
  chunk_overlap: 64
  device: cpu                                          # cpu | mps | cuda
  offline: false                                       # flip true to skip every HF Hub call
```

Available template keys: `{author_last}`, `{author_last_first}`, `{year}`, `{title_slug}`, `{doi_slug}`, `{journal_slug}`. All resolve `~` and env vars.

## How it works

```
        +-----------+        +---------------+        +---------------+
PDF --> | pdf_meta  | -----> | Crossref /    | -----> | PaperMatch    |
        | (sniffs   |  DOI / | arXiv /       |        | (canonical    |
        | DOI/ISBN) |  ISBN  | Open Library  |        | dataclass)    |
        +-----------+        +---------------+        +-------+-------+
                                                              |
                          +-----------------------------------+
                          v
              +-----------+----------+        +--------------+
              | SQLite library       |        | path render  |
              | papers / authors /   | -----> | + copy/move/ |
              | tags / fk constraints|        | symlink PDF  |
              +-----------+----------+        +--------------+
                          |
                          | needs_reindex=0/1
                          v
              +-----------+----------+        +--------------+
              | extract -> chunk ->  | -----> | Chroma       |
              | embed (sentence-     |        | persistent   |
              | transformers)        |        | collection   |
              +----------------------+        +--------------+
```

A few decisions worth flagging:

- **Weighted DOI scoring.** A DOI on page 0 or in PDF metadata counts more than a DOI buried in the bibliography. A confidence threshold prevents a body-citation from being mistaken for the paper's own identifier when you import in `--quiet` mode.
- **ISBN reach.** ISBN-labelled hits on the colophon page (often page 5 or 6 of a textbook) are picked up by scanning pages 0..5 plus the last page; bare 13-digit numbers require a label nearby to avoid false positives.
- **Migrations are additive and idempotent.** `connect()` always runs `init_db()`; new columns + partial unique indexes are applied with `ALTER TABLE` on existing databases.
- **Chroma never gates the import.** If embedding fails, the SQLite row commits with `needs_reindex=1` so you can fix it with `book reindex` later.

## Project layout

```
src/books/
  cli.py                Typer app, subcommand registration
  config.py             confuse helpers (typed accessors)
  config_default.yaml   shipped defaults
  paths.py              slugify + path templating + place_pdf()
  db.py                 schema, migrations, CRUD
  query.py              flag/positional CLI -> parameterised SQL
  importer.py           sniff -> lookup -> confirm -> store -> index
  interactive.py        rich-based prompts
  commands/             one file per CLI subcommand
  metadata/
    pdf_meta.py         DOI/arXiv/ISBN sniffing + scoring
    crossref.py         Crossref REST client
    arxiv.py            arXiv Atom client
    openlibrary.py      Open Library ISBN client
    models.py           PaperMatch / Author dataclasses
  index/
    extract.py          PyMuPDF -> [(page_no, text)]
    chunker.py          semantic-text-splitter wrapper
    embedder.py         Embedder Protocol + sentence-transformers impl
    chroma.py           PersistentClient wrapper
    indexer.py          orchestration + lazy embedder singleton
tests/                  pytest suite (48 tests, runs in <2s)
```
